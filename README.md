Каждый лендинг — **один HTML-файл со всем внутри**. Никаких сборщиков, npm-зависимостей, отдельных CSS/JS-файлов.

## Как открыть

### Прямо в браузере

```bash
open ~/Codex/landings/index.html
```

### Локальный сервер (рекомендуется)

```bash
cd ~/Codex/landings
python3 -m http.server 8000
```

И открыть `http://localhost:8000` в браузере.

## Технические принципы

- **Один HTML-файл на лендинг.** CSS — в `<style>`, JS — в `<script>` внизу.
- **Внешние ресурсы:** только Google Fonts CDN и плейсхолдеры фото с picsum.photos (по seed-URL для стабильности).
- **Адаптив:** работает от 320px. Мобильное меню-бургер через inline JS.
- **Демо-формы.** Все формы визуальные: `<form onsubmit="return false">`, кнопки `type="button"`. Подключение бэкенда — отдельная задача.
- **Минимум JS.** До 30 строк ванильного JS на лендинг. Только: аккордеоны, табы, обратный отсчёт, мобильное меню, счётчик товара.
- **Иконки** — inline SVG.
- **Комментарии** — на русском, к крупным блокам и функциям JS.

## Доработка под клиента

Каждый лендинг — самодостаточный шаблон. Чтобы адаптировать под реальный проект:

1. Скопируйте папку нужного лендинга
2. Замените текст и плейсхолдеры на реальный контент
3. Замените `picsum.photos` URLs на свои фото
4. Подключите формы к бэкенду (убрать `onsubmit="return false"`, добавить `action` и `method`)
5. Замените заглушки реквизитов в футере
